# Паттерн команда

Свою реализацю монад я выполнил ближе к command.

Сейчас выход прошлой программы будет сразу поступать на вход следующей, но в формулировке будто бы просят избавиться от стейта внутри монады, а оставить cleaner только как входящий и исходящий элемент.
Надо просто убрать поле cleaner и все попрет.
А OutMessage надо добавлять в List<OutMessage> и выводить в лог только при получении его.

### Плюсы
* Команды сформированы однажды по входным данным и могут быть выполнены при необходимости.

### Минусы
* fold в функциональном стиле выполнить не получилось. Не удалось засунуть это в синатуру reduce от StreamAPI.

### Рефлексия по решению
Оказывается сделал все верно по сигнатурам.   
+Не имелось в виду, что надо сделать в функциональном стиле, было достаточно так же пробежаться циклом, а не использовать API из функционального Stream.

В реализации каждой комманды надо было оставить только поле под лямду.  
А передавать реализацию в CommandExecutor.   
Тогда вся завязка на конкретную реализацию осталось бы в CommandExecutor, а не была размазана в командах, подобно тому, как это сделано в монадах.
