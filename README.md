# Clear_Architecture
Практика в Архитектуре на примере управления роботом уборщиком

## Рефлексия по решению

Начал с мышления письмом и понял, что могу быстренько выделить основные АТД по Бертрану Мейеру, т.к. это наиболее знакомый мне стиль проектирования.
Ввел ограничения для АТД, например, то что метры должны быть положительные
После движения робот не может оказаться за пределами поля
То что угол должен быть в пределах 360 градусов и внутри программы будет положительным (-90 преобразую в 270 на входе).
Интересно что даже в наивном стиле выделили Enum, это уже не мало :).

Саму программу после выделения всех АТД и введения ограничений было уже проще реализовать, т.к. все для этого было.
IDE открыл только когда реализовал все классы в блокноте. Так не распыляюсь на более богатый интерфейс IDE и возможности запуска кода.
Поправить потребовалось буквально 5-20 строк геттеров и добавить импорты.
Была одна логическая ошибка, когда использовал Map вместо List<Pair<String, Object>>.
Так же смущает, что приходится выполнять каст типа перед передачей аргумента в метод.
