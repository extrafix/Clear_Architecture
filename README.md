# Clear_Architecture
Практика в Архитектуре на примере управления роботом уборщиком

## Функциональный DI
### Решение по передаче каждой реализаци отдельной функцией
В Java я могу использовать функциональные интерфейсы, то есть те, для реализации которых смогу передавать просто лямду, не создавая явно класс.   
Писать реализации в вызовах я не хочу, поэтому использую плюсы модульнуго стиля и каждую функцию реализую в своем файле, который будет реализацией функционального интерфейса.  
Затем, эти реализации я смогу подавать в метод, в котором до этого вызывал реализации статических методов.  

Вопросы:
Как мне разделить функции, которые предпологают входящие аргументы и которые их не предполагают?  
Самый простой вариант, просто передавать аргумент всегда, а в реализации уже не использовать, но это начнет путать код.  
Сделай сначала так, а потом придумай вариант лучше.  
Хотелось бы избавиться от вызова методов статического класса.  
Реализации я по прежнему могу брать из статического класса, но переместить это в конфигурацию из кода логики.
