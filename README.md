# Монады состояний 

Думал о таком при реализации функционального стиля.  
Основные сложности были, надо ли указывать аргументом объект Cleaner, но т.к. он содержится в монаде, как в Stream содержатся элементы из списка - все оказалось просто.
Просто после каждого обращения создается новая монада с новым объектом Cleaner и от нее можем вызвать любые другие ее методы.

### Плюсы
* Красивый синтаксис.

### Минусы
* Реализация не lazy.
